Introduction & Context

💳

Application Focus

Online Payment Application (UPI-based) enabling quick, easy, and secure money transfers between users.

👥

Target Users

Students, working professionals, and general smartphone users requiring fast, reliable payment solutions.

🎯

Design Purpose

Existing payment apps suffer from confusing navigation, lack of feedback, and high chances of user errors.

This redesign focuses on improving usability by applying heuristic principles and reducing cognitive load to enhance user decision-making and interaction flow.
02 / 16

🔍
Heuristic Evaluation Summary

The following Nielsen's heuristics were used as the foundational framework for evaluating the existing payment application:

📡

Visibility of System Status

The system should always keep users informed about what is going on through appropriate feedback within reasonable time.

🌍

Match Between System and Real World

The system should speak the users' language with words, phrases, and concepts familiar to the user.

↩️

User Control and Freedom

Users often choose system functions by mistake and need a clearly marked "emergency exit" to leave the unwanted state.

📐

Consistency and Standards

Users should not have to wonder whether different words, situations, or actions mean the same thing.

🛡️

Error Prevention

Even better than good error messages is a careful design which prevents a problem from occurring in the first place.

03 / 16

⚠️
Identified Usability Issues

  • 1
    Lack of proper confirmation before payment — Users can accidentally complete transactions without adequate verification steps, leading to unintended money transfers.
  • 2
    High chances of entering incorrect amount — The amount input interface lacks validation checks, decimal handling, and visual confirmation of entered values.
  • 3
    Lack of system feedback during processing — Users are left uncertain about transaction status during payment processing with no loading indicators or progress updates.
  • 4
    Confusing navigation and interaction flow — Inconsistent screen transitions and unclear information architecture make task completion unnecessarily complex.
  • 5
    No effective error prevention mechanisms — The system fails to proactively identify and prevent common user errors before they result in failed transactions.
Severity: High — Can lead to financial errors and user frustration
04 / 16

🎯
Problem Framing

Three core problems were identified through user research and heuristic evaluation that directly impact the payment experience:

1

Input & Validation Errors

Users make errors due to unclear information and lack of validation. Incorrect amounts, wrong recipients, and missing confirmations create critical failure points.

2

Navigation Confusion

Users experience confusion due to poor interaction flow. Unclear screen hierarchies and inconsistent navigation patterns disrupt task completion.

3

Feedback Uncertainty

Users feel uncertainty due to lack of feedback. Absence of system status updates during transactions creates anxiety and trust issues.

05 / 16

🧠
Cognitive Impact Analysis

The identified usability issues create significant cognitive burdens that degrade the overall user experience:

🧩 High Cognitive Load — Excessive mental effort required to navigate and complete transactions
💭 Increased Memory Dependency — Users must remember steps rather than recognize options
❓ Poor Decision-Making — Unclear information leads to hesitant and error-prone choices
👁️ Low Attention Focus — Distracting interface elements divert attention from critical tasks
These issues negatively affect user experience and increase the chances of mistakes, particularly in high-stakes financial transactions where errors carry real monetary consequences.
06 / 16

🔗
Heuristic Justification

Each identified problem was mapped to specific heuristic violations to establish a clear rationale for design decisions:

Usability Issue Violated Heuristic Design Justification
No feedback during processing Visibility of System Status Users cannot determine if the system is working, causing anxiety and premature action repetition.
No confirmation before payment Error Prevention Critical financial actions require confirmation barriers to prevent accidental transactions.
Confusing navigation flow Consistency and Standards Inconsistent patterns force users to relearn interactions, increasing cognitive load and errors.
Hard to find frequent contacts Recognition over Recall Users must recall contact details rather than recognize them visually, burdening working memory.
